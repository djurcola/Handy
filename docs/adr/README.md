# Architecture Decision Records

This directory contains ADRs (Architecture Decision Records) for Handy.

## Index

No ADRs have been recorded yet.

### Adding a new ADR

1. Create a new file: `NNNN-short-title.md` (e.g., `0001-sequential-transcription.md`).
2. Use this template:

   ```markdown
   # ADR-NNNN: Title

   **Status:** Proposed | Accepted | Deprecated | Superseded by ADR-NNNN
   **Date:** YYYY-MM-DD

   ## Context

   What is the issue or situation that motivates this decision?

   ## Decision

   What is the change that we're proposing and/or doing?

   ## Consequences

   What becomes easier or more difficult to do because of this change?
   ```

3. After creating the ADR, add a link in `docs/architecture.md` under the "Key Decisions" section.
